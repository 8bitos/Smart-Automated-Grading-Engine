---

### /backend/
- **Fungsi:** Berisi kode untuk aplikasi backend yang dibuat menggunakan bahasa Go.
- **Isi Penting:**
    - `main.go`: Titik masuk utama aplikasi backend. Saat ini berisi server web sederhana yang menyediakan endpoint `/api/hello`.
    - `go.mod`: File definisi modul Go untuk manajemen dependensi.
    - `db/`: Direktori yang khusus menangani operasi database, termasuk migrasi.

---

### /backend/db/migration/
- **Fungsi:** Menyimpan file-file migrasi database yang digunakan untuk membuat dan mengelola perubahan skema database PostgreSQL secara terstruktur dan terverifikasi.
- **Isi Penting:**
    - `000001_init_schema.up.sql`: Berisi perintah SQL (`CREATE TABLE`, `CREATE TYPE`, dll.) untuk membuat semua tabel, tipe data, dan relasi awal di database.
    - `000001_init_schema.down.sql`: Berisi perintah SQL (`DROP TABLE`, `DROP TYPE`, dll.) untuk membatalkan (rollback) migrasi `up.sql` jika diperlukan.
- **Cara Kerja Migrasi:** Dengan bantuan alat seperti `golang-migrate/migrate`, file-file ini dijalankan secara berurutan untuk memperbarui skema database. Setiap perubahan skema akan diwakili oleh sepasang file `up.sql` dan `down.sql` baru.

---

### /frontend/
- **Fungsi:** Berisi kode untuk aplikasi frontend yang dibuat menggunakan Next.js (React framework).
- **Isi Penting:**
    - `src/app/page.tsx`: Komponen halaman utama yang akan diakses pengguna. Saat ini, halaman ini mengambil dan menampilkan pesan dari backend.
    - `package.json`: Mengelola semua dependensi (packages) JavaScript dan mendefinisikan skrip untuk menjalankan aplikasi (misalnya, `npm run dev`).
    - `node_modules/`: Direktori tempat semua dependensi diinstal. (Folder ini diabaikan oleh Git).

---

### .gitignore
- **Fungsi:** File ini berisi daftar file dan folder yang harus diabaikan oleh Git. Ini mencegah file-file yang tidak perlu (seperti `node_modules`, file environment `.env`, atau file hasil kompilasi) masuk ke dalam repository GitHub.

---

### docker-compose.yml
- **Fungsi:** File konfigurasi untuk Docker Compose. File ini mendefinisikan dan mengelola layanan-layanan pendukung yang dibutuhkan oleh aplikasi, yaitu:
    - **postgres:** Database relasional untuk menyimpan data terstruktur.
    - **redis:** In-memory cache untuk menyimpan data sementara dan mempercepat respon.
    - **elasticsearch:** Vector database untuk fitur pencarian semantik (RAG).
- **Cara Kerja:** Dengan satu perintah (`docker-compose up`), semua layanan ini akan berjalan secara otomatis di dalam container Docker yang terisolasi.

---

### database.json
- **Fungsi:** Berisi definisi skema database dalam format JSON yang mudah dibaca. File ini berfungsi sebagai "blueprint" atau dokumentasi formal untuk semua tabel, kolom, tipe data, dan relasi dalam database PostgreSQL kita, yang kemudian digunakan untuk membuat file migrasi SQL.

---

### Struktur.txt
- **Fungsi:** File ini (yang sedang Anda baca) memberikan penjelasan mengenai struktur proyek untuk referensi di masa depan.
